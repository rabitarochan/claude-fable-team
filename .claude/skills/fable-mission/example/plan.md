## 前提

- Node.js 22 がローカルで利用可能
- データ量は個人利用規模(SQLite で十分)。崩れたら DB 選定から再計画

## 主要な設計判断

| 判断 | 採用案 | 理由 | 採らなかった案 |
|---|---|---|---|
| DB アクセス | better-sqlite3 | 同期 API でテストが単純になる。規模に十分 | Prisma(小規模には過剰。依存が深い) |
| 認証方式 | API キー(ヘッダ) | 発注者が「簡単でいい」と明言 | JWT(スコープ外の本格認証に寄りすぎ) |
